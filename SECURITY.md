# Security Policy

## Supported Versions

Security fixes are applied to the default branch and latest container image tag.

## Reporting a Vulnerability

Do not open public issues for sensitive vulnerabilities.

Report vulnerabilities to: harry.valdez@gmail.com

Include:

- Vulnerability type and impact
- Reproduction steps
- Affected versions/commit hashes
- Suggested remediation (if available)

## Disclosure Process

1. Maintainers acknowledge receipt within 2 business days.
2. Maintainers triage severity and impact.
3. Fix is developed and validated.
4. Patch is released and advisory shared when appropriate.

## Response Targets

- Acknowledgement: 2 business days
- Initial triage: 5 business days
- Remediation target: based on severity and exploitability
