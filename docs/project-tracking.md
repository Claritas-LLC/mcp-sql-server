# Project Tracking

## Issue Lifecycle

- `needs-triage`: New issue requires maintainer review
- `ready`: Scoped and accepted for implementation
- `in-progress`: Actively being worked
- `blocked`: Waiting on dependency/external input
- `done`: Merged and validated

## Recommended Project Board Columns

- Backlog
- Ready
- In Progress
- Review
- Done

## SLA Targets

- New issue triage: within 2 business days
- High-severity bug acknowledgement: within 1 business day
- PR first review: within 2 business days

## Milestones

- Use milestones for grouped deliveries (for example, `v1.1-governance`, `v1.2-security`).
- Assign issues to a milestone when scope and target release are known.
